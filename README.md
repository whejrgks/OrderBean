## 🎯 제품 개요

### 문제 정의

- 카페 평점은 높지만 개인의 커피 취향과 맞지 않는 실패 경험 반복
- 커피 맛(산미, 로스팅, 바디감 등)에 대한 정보가 구조화되어 있지 않음
- 목적(공부, 미팅 등)과 커피 취향을 동시에 만족하는 카페 탐색의 어려움

### 목표

- 취향 기반 카페 선택 성공 확률 증가
- 커피 중심 데이터 표준화 및 추천 정확도 향상

### 대상 사용자

- 커피 취향이 어느 정도 형성된 일반 소비자
- 직장인, 학생, 프리랜서 (카공·미팅 목적)
- 여행 중 로컬 카페를 탐색하는 사용자

## ✨ 주요 기능

### FR-1. 커피 취향 프로필 관리
- 사용자는 취향 설문을 통해 개인 커피 취향 프로필을 생성·수정할 수 있습니다
- 관리자는 취향 항목 및 가중치를 관리할 수 있습니다

### FR-2. 취향 기반 카페 지도 검색
- 사용자는 지도에서 카페를 취향 일치도 기준으로 탐색할 수 있습니다
- 관리자는 카페 위치 및 기본 정보를 관리합니다

### FR-3. 취향 일치도 기반 카페 추천
- 시스템은 사용자 취향과 카페 데이터를 비교하여 일치도를 계산합니다
- 추천 결과에는 점수와 추천 사유가 함께 제공됩니다

### FR-4. 커피 맛 리뷰 및 태그
- 사용자는 카페 및 메뉴에 대해 맛 중심 리뷰를 작성할 수 있습니다
- 관리자는 리뷰 태그를 표준화하고 부적절한 리뷰를 관리합니다

### FR-5. 관리자 운영 대시보드
- 관리자는 사용자, 카페, 리뷰를 통합 관리할 수 있습니다
- 서비스 이용 현황 및 추천 성능을 모니터링합니다

## 💎 핵심 가치

- **개인 커피 취향 기반 추천**: 사용자의 취향을 정밀하게 분석하여 맞춤형 추천 제공
- **지도 중심 직관적 탐색**: 지도 기반 UI로 주변 카페를 쉽게 탐색
- **투명한 추천 시스템**: 추천 이유가 설명 가능한 명확한 추천 로직

### 주요 디렉토리 설명

- **frontend/**: 사용자 인터페이스 및 클라이언트 로직
  - 지도 기반 카페 탐색 UI
  - 취향 설문 인터페이스
  - 리뷰 작성 및 조회 기능
  - 관리자 대시보드

- **backend/**: 서버 사이드 로직 및 API
  - RESTful API 엔드포인트
  - 취향 일치도 계산 엔진
  - 사용자 인증 및 권한 관리
  - 데이터 검증 및 보안

- **database/**: 데이터베이스 스키마 및 마이그레이션
  - 사용자, 카페, 리뷰 데이터 모델
  - 취향 프로필 데이터 구조
  - 인덱스 및 성능 최적화

- **tests/**: 테스트 코드
  - 단위 테스트 (비즈니스 로직)
  - 통합 테스트 (API 엔드포인트)
  - E2E 테스트 (사용자 시나리오)

## 🎬 사용자 시나리오

### 커피 취향 프로필 생성

```
Given 사용자가 로그인한 상태이고
And 취향 설문 페이지에 접속하며
When 사용자가 산미를 "높음"으로 선택하고
And 로스팅을 "라이트"로 선택한 후
And "설문 완료" 버튼을 클릭하면
Then 커피 취향 프로필이 저장되고
And 취향 요약이 표시된다
```

### 취향 기반 카페 지도 검색

```
Given 사용자가 취향 프로필을 보유하고
When "취향 일치도 순" 필터를 선택하면
Then 카페가 지도에 점수와 함께 표시된다
```

### 취향 일치도 추천

```
Given 사용자 취향과 카페 데이터가 존재하고
When 카페 상세 페이지에 접속하면
Then 취향 일치도와 추천 사유가 표시된다
```

### 리뷰 작성

```
Given 사용자가 로그인 상태이고
When 맛 태그와 리뷰를 입력 후 등록하면
Then 리뷰가 저장되고 공개된다
```

## 📊 비기능 요구사항

### 성능
- 지도 검색 결과는 3초 이내 로딩
- 취향 일치도 계산 결과는 1초 이내 표시

### 확장성
- 카페 100,000개 이상에서도 검색 성능 유지
- 추천 알고리즘은 교체 가능하도록 모듈화

### 보안
- 사용자 비밀번호 암호화 저장
- 관리자 기능은 권한 기반 접근 제어

### 사용성
- 취향 설문은 평균 1분 이내 완료 가능
- 모바일 환경에서도 한 손 조작 가능

### 접근성
- 키보드만으로 주요 기능 접근 가능
- WCAG 색상 대비 기준 준수

### 데이터 품질
- 리뷰 태그는 표준화된 값만 허용
- 악성/중복 리뷰 탐지 기능
